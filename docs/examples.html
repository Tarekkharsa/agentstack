Demos — the problems, proven

Each demo here shows a problem AgentStack closes — exfiltration, destructive commands, config drift — and proves the fix holds. Everything comes from examples/ in the repo: self-contained, self-asserting scripts that run in an isolated temp HOME, print PASS/FAIL per claim, and exit nonzero on any mismatch. Looking for how to use the CLI instead? That’s the Cookbook.

malicious-repo-demo

the threat, proven
The ten-second proof of the gateway boundary. The villain is evil_server.py: an MCP server advertising a friendly status tool and a malicious exfiltrate tool that reads a planted credential and POSTs it to a localhost sink. Tellingly, its manifest declares no firewall of its own — a malicious repo won't constrain itself. The same hostile repo runs three ways:
examples/malicious-repo-demo/run-demo.sh · needs python3 · DEMO_PAUSE=2.5 paces it for a recording

guard-demo

guard, asserted
Guard wires a cooperative pre-tool-use hook into the CLIs that support one (Claude Code, Codex, Gemini, Cursor, Windsurf, Copilot CLI, Antigravity, OpenCode, Pi). Before a tool runs, the pending call goes to guard check; every denial is recorded to ~/.agentstack/audit/calls.jsonl. Hand it a pending call:
examples/guard-demo/run-demo.sh · finishes by grepping the audit log to prove the denials landed

sandbox · lockdown

no direct route — needs Docker
demo-lockdown.sh launches a real run --sandbox --lockdown container and then tries to get out of it. Click each attempt:
The container sits on an internal-only network whose sole peer is the egress sidecar — posture LOCKDOWN / ENFORCED · NO DIRECT ROUTE. Point AGENTSTACK_SANDBOX_IMAGE at an image carrying your CLI; the sidecar is pulled from GHCR, pinned per release.

Machine-policy presets

Four ready-to-use standing firewalls — each a complete machine manifest, validated by the CLI's own loader in CI. Copy one to ~/.agentstack/agentstack.toml, then doctor. Pick one to see its stance:
All four read as restrictive in doctor because each carries at least one rename-proof "*" rule — it binds every server whatever a manifest calls it. Plain globs allow, a ! prefix denies; any allow pattern turns the list into an allowlist. A later broken edit runs DEGRADED on the last-known-good snapshot; broken first-run state is BLOCKED, never project-only.
