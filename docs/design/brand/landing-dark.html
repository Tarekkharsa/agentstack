Build, govern, and run
your agent stack.

One manifest for all 13 agent CLIs — servers, skills, instructions, and secrets, defined once. Nothing runs until you trust it, and nothing trusted runs unobserved.

Get started · 10 min → Install
the whole stack, governed 
$ agentstack apply            # one manifest → 13 CLIs, in sync
$ agentstack guard install    # blocks rm -rf / .env accidents in 9 CLIs
$ agentstack trust .          # cloned repos stay inert until you consent
$ agentstack run --lockdown   # no route out but the egress firewall
$ agentstack report           # every brokered call, on the record

01 · You juggle several CLIs

One manifest keeps them all in sync

Claude Code at work, Cursor for edits, Codex in CI — and every MCP server configured three different ways. Define it once; agentstack apply renders each CLI's native config, with secrets left as per-machine references.

one manifest → .mcp.json · config.toml · mcp.json 
$ cat .agentstack/agentstack.toml   # one committed source of truth
  [servers.github]
  command = "npx"  args = ["-y", "@modelcontextprotocol/server-github"]
  env = { GITHUB_PERSONAL_ACCESS_TOKEN = "${GITHUB_TOKEN}" }
 
$ agentstack apply --write
  Claude Code   + .mcp.json                ✓ wrote 1 server
  Codex CLI     + .codex/config.toml       ✓ wrote 1 server
  Cursor        + .cursor/mcp.json         ✓ wrote 1 server
  instructions  → CLAUDE.md + AGENTS.md    ✓ compiled
 
  ✓ same server, three native shapes — the manifest never holds the token

Why this exists

Your agents run whatever your repos tell them to.

Every skill, MCP server, and agent config you adopt is unreviewed code plus instructions, wired into a process that holds your credentials, your shell, and the network.

One clone away

A repo can declare MCP servers, and the moment an agent session starts, they want to run. → Here, a clone is inert until you approve its consent surface.

One prompt away

An injected instruction can steer a well-behaved agent into using a legitimate tool against you. → Here, your machine policy fences tools, secrets, and egress.

Invisible afterwards

Most setups can't answer the first question a reviewer asks: what did the agent actually call? → Here, every brokered call and denial lands in the call audit.

Honestly scoped

No tool can make agent use risk-free: hosts you allow stay allowed. → That's why every mode carries an honest posture label.
