Cookbook — AgentStack, by example

Twenty-six copy-paste recipes for using the CLI, building from a one-line manifest to sandboxed, governed execution. The manifest is the source of truth; generated native configuration is compiled output. Watching the security claims proven instead? That's the Demos page.

How to read the evidence: unless a recipe is marked runnable fixture, treat its code as a focused, CLI-validated recipe rather than a self-asserting end-to-end proof.

Configure

$ agentstack apply

Get a server into your tools, organize capabilities by task, reuse them everywhere, extend the tool itself.

1
The smallest manifest
One MCP server, one CLI — the whole thing at its simplest. The CLI reads its rendered file once at launch; change the manifest, re-run apply, restart the CLI.

# .agentstack/agentstack.toml
version = 1
[servers.time]
type = "stdio"
command = "uvx"
args = ["mcp-server-time"]
[targets]
default = ["claude-code"]
$ agentstack apply --write # writes Claude Code's native .mcp.json for you
Result → Claude Code now has the official Time reference server, generated from your manifest — you never hand-edited its config.

3
Keep secrets out of the file
Never paste a token in. Reference it as ${NAME}; it resolves per machine through the secret chain: environment, varlock, OS keychain, then .env.
a secret, resolved at runtime
${GH_PAT}
manifest holds the ref only
→
OS keychain
secret set — real value
→
per call
gateway injects it
→
server
gets the token
note → the committed file never contains a real secret; an unresolvable ref blocks the write, never leaves a placeholder.
[servers.github]
type = "http"
url = "https://api.githubcopilot.com/mcp/"
headers = { Authorization = "Bearer ${GH_PAT}" } # a reference, never the value
$ agentstack secret set GH_PAT # stored in the OS keychain, not a file
$ agentstack doctor # flags a referenced secret that's missing
Result → The committed manifest is safe to share. A missing secret blocks the write rather than leaking a placeholder into live config.
4
An HTTP server with auth
Remote MCP servers use type = "http", a url, and headers. Run doctor --live before an agent depends on it: a stale token surfaces now, with the exact transport error, instead of mid-session.
[servers.kibana]
type = "http"
url = "https://kibana-mcp.example.com/mcp"
headers = { Authorization = "Bearer ${KIBANA_TOKEN}" }
$ agentstack doctor --live # real MCP handshake: name + tool count, or the exact error
5
A native key for one CLI
When one CLI needs a native key another doesn't, pass it through verbatim under extra.<cli> — copied through, never interpreted.
[servers.filesystem]
type = "stdio"
command = "npx"
args = ["-y", "@modelcontextprotocol/server-filesystem", "./"]
[servers.filesystem.extra.codex]
startup_timeout_sec = 20 # Codex sees this; the others don't
Result → Only Codex's generated config carries startup_timeout_sec; every other CLI's stays untouched.
6
Add skills
Skills are reusable instruction/tool bundles. Point at a local directory — relative paths anchor at the manifest's directory (.agentstack/). Skills activate through profiles (next), not apply.
[skills.sql-review]
path = "./skills/sql-review" # lives at .agentstack/skills/sql-review
7
Task-specific profiles
A profile is a named bundle of servers + skills you switch on for a kind of work. Fewer tools on the wire means fewer tokens billed every turn (see §21).

[profiles.backend]

[profiles.docs]
servers = ["github"]
skills = ["writing-style"]
$ agentstack use backend --write # activate the backend profile
Result → Load only what the task needs — others stay off.

Share house rules across CLIs
Compile shared instructions into each CLI's native file (CLAUDE.md, AGENTS.md…) from one source. Edit it, re-run instructions --write, and all targets update together.
[instructions.house_rules]
path = "./house-rules.md"
# targets = ["*"] # optional: which CLIs get it (default: all)
26
Native extensions
Manage a harness's native executable add-ons — pi's TypeScript extensions, OpenCode's JS plugins — the way [skills.*] manages skill dirs. The highest-risk capability agentstack delivers: the code runs inside the harness at full user permission, outside every policy ceiling. agentstack pins and delivers the bytes; it never executes or governs them at runtime.
[extensions.checkpoint]
description = "Git checkpoint on every agent turn"
path = "./extensions/checkpoint" # or: git = "…", rev = "…", subpath = "…"
target = "pi" # exactly one adapter id — no targets list, no "*"
Result → Strictly pinned: a symlink is a hard error; unpinned executable content blocks until agentstack lock accepts it; an untrusted or drifted project renders zero extension bytes. Renders today for pi and OpenCode.

The everyday loop
The four commands you'll actually run. Two habits worth keeping: agentstack with no arguments tells you the single next step for the directory you're in; doctor --ci fails a build on errors, drift, policy, or unsafe content.

14
The central library
Install a capability once into your machine-wide library (~/.agentstack/lib), then reference it by name from any project — nothing copied, and every add is content-scanned before it lands.
$ agentstack lib add server github # once, machine-wide, content-scanned
$ agentstack add github # any project: reference by name
15
Sync the library across machines
Version the library as a git repo with lib sync — a fail-closed gate scans every server field and the outgoing commits for secrets before pushing. A literal secret, or a definition it can't parse, blocks the sync.
$ agentstack lib sync # git-versioned; secret-scanned before every push
16
Versioned vendor packs
Install a pinned bundle of MCP servers, skills, and house rules in one shot — policy-gated and content-scanned first, with the lockfile recording exactly what you got. Upgrading is just the next tag.
$ agentstack add pack github.com/acme/agent-pack@v2.1.0 # pinned, scanned, locked
24
Extend it — add a CLI in one file
Support a new CLI with a single YAML descriptor — no rebuild. A broken drop-in is skipped with a warning, never fatal.
$ cp crates/adapters/descriptors/codex.yaml my-adapter.yaml
# edit it, then:
$ agentstack adapters validate my-adapter.yaml
$ cp my-adapter.yaml ~/.agentstack/adapters/ # picked up on next run
$ agentstack adapters list # marks which adapters are yours
25
The personal layer & app-owned servers
Machine-wide instructions that merge beneath every project without landing in any repo; and letting an app's own config be the source of truth. When the app rewrites its own entry (a self-update), apply refreshes the manifest and fans the fresh values out to every other CLI instead of reverting the app.
$ agentstack init --global # a home for your machine-wide rules
[servers.codex-native]
owner = "codex" # codex owns this entry; apply follows it
Result → Machine policy survives bad edits: a validated policy continues from a secret-free last-known-good snapshot and reports DEGRADED; first-run corruption reports BLOCKED.

10
The MCP tool firewall
Restrict which tools a server may expose. Plain globs allow; a leading ! denies. Any allow pattern makes the list an allowlist. A denied tool is invisible — filtered from discovery — and refused with the rule named if called anyway.
the firewall, on every call
agent
calls a tool
→
gateway
[policy.tools]
→
✓ get_issue
allowed
→
✗ list_secrets
refused, rule named
note → a denied tool is refused before it reaches the server, and hidden from discovery entirely.
[policy.tools]
github = ["get_*", "list_*", "!list_secrets"] # read-only, minus one tool
Result → agentstack explain github shows the effective policy.
11
The machine layer — rules no repo can loosen
Your standing rules live in ~/.agentstack/agentstack.toml and are checked before any project's. A call must pass both layers — the effective policy is the intersection, never more permissive than your machine layer.
policy is an intersection
machine policy
your standing rules
→
∩ repo policy
can only add, not loosen
→
effective
never more permissive than the machine layer
# ~/.agentstack/agentstack.toml — applies to every project on this machine
[policy.tools]
"*" = ["!delete_*"] # no server, under any name, may delete_*
github = ["get_*", "list_*"] # servers NAMED github are read-only
Result → The "*" key is rename-proof — it binds every server whatever a manifest calls it.

Zero files: the trust gate
runnable fixtures
Skip rendered files entirely. Register the gateway once; every trusted repo then serves its own servers live — an unreviewed repo is inert. Edit the manifest or lock and it drops back to inert until you review again.
clone → inert → trusted → audited
clone
inert — nothing spawns
→
trust .
review, then pin bytes
→
firewalled
servers serve live
→
audited
every call logged
note → edit the manifest or lock and it drops back to inert until you review again.
$ agentstack gateway connect --all --write # register the gateway (once)
$ git clone <some-repo> && cd <some-repo>
$ agentstack mcp --auto-project # an agent asks what's here → nothing (untrusted)
$ agentstack trust . # SEE what it declares, then authorize
$ agentstack trust .
▶ demo: runs `python3 ./server.py` # exactly what it would run
✓ trusted at sha256:945b4b… # editing the manifest/lock re-gates it
# now its servers serve live, through the firewall:
agent → demo.echo ✓ ok
agent → demo.secret_read ✗ denied [policy.tools]
Result → Every brokered call lands in ~/.agentstack/audit/calls.jsonl. After trust, an agent can fence one MCP connection to a profile with agentstack_lease_open — no .mcp.json, skill folder, or session entry written; lease_freeze proposes a manifest profile from what the task actually used.
19
The other policy dimensions
The same two-layer intersection and "*" grammar apply beyond tools and egress. Enforcement depends on execution mode: host/gateway filesystem guards are cooperative harness hooks; sandbox/lockdown use coarse workspace mounts.
[policy.secrets]
github = ["GH_*"] # server named github may resolve only GH_* refs
[policy.filesystem] # bundle-global path globs
read = ["./src/**", "./docs/**"]
write = ["./out/**", "!**/.env"] # never write a .env, even under an allowed root

Run confined

$ agentstack run --sandbox [--lockdown]

Confine the run: containers whose only way out is the audited egress proxy, and governed code execution.

18
Sandboxed runs with egress policy
runnable · Docker
Run the agent in a container with no host route and no internet — its only way out is the egress-proxy sidecar, which enforces your machine [policy.egress]. Confinement is topological: ignoring the proxy reaches nothing.
the only way out is the proxy
container
no host route, no net
→
egress sidecar
[policy.egress]
→
✓ api.github.com
allowed host
→
✗ evil.example
refused
note → ignoring the proxy reaches nothing — it's the network shape, not an honor system.
# ~/.agentstack/agentstack.toml — same glob grammar as the tool firewall, over hosts
[policy.egress]
"*" = ["!*"] # deny everything by default
github = ["api.github.com:443"] # a server named github may reach only this
$ agentstack run claude-code --sandbox --lockdown --profile backend
$ agentstack report run <run-id> # the run's flight recorder: every egress decision
Result → Filters by host, requires a TLS connection's SNI to match the dialed host (no domain fronting), and refuses names resolving to loopback/private/link-local/metadata addresses (no SSRF). Needs Docker + --features sandbox.

Experimental: governed TypeScript
tools_execute lets one bounded TypeScript program call a small, exact set of MCP tools through the gateway — as a non-root process in a read-only, resource-limited Docker container with no workspace, credentials, package install, or direct network. Off by default; only the machine owner enables it.
governed code, sandboxed
TS program
exact tool grant
→
sandbox
no net, no creds
→
gateway
re-checks each call
→
result
attributed in report
# ~/.agentstack/agentstack.toml
[experimental]
tools_execute = true
# optional machine-owned ceilings; a request may only narrow them
[experimental.tools_execute_limits]
timeout_ms = 30000
max_calls = 40
max_output_bytes = 131072
Result → Every allowed call is re-checked by the gateway and attributed in agentstack report. Docker-only; never a host-execution fallback.

Observe

$ agentstack report calls

See what happened, and what it costs: the audit trail and the passive wire proxy.

20
See what happened — audit, analyze, report
Every brokered call is recorded (digests and outcomes, never argument values or secrets). Turn that into insight.
$ agentstack report calls --since 7 # recent brokered calls: tool · outcome · latency
$ agentstack report run <run-id> # per-run tree, incl. sandbox egress decisions
$ agentstack report calls # what you actually call; flags installed-but-unused
21
See what your tools cost
Every tool/server/skill you load is re-billed as input tokens every turn. Measure it with the passive wire proxy — observe only, cache stays warm.
$ export ANTHROPIC_BASE_URL=http://127.0.0.1:8787
$ agentstack proxy # in one shell, then use your agent normally
$ agentstack report wire # ranks tokens/turn vs. how often each was called
Result → A server that's expensive but never used is flagged drop / lazy — data-driven pruning that closes the loop with the profiles you manage.
Our default recommendation: for interactive work in an MCP-capable harness, commit the manifest + lock, connect the gateway once, trust each reviewed repo, and open the smallest useful profile as an MCP lease. Choose native rendering only when the harness must read native files; add lockdown when the agent process needs isolation.
