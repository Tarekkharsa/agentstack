LESSON 1 · WELCOME

Cloning a repo shouldn't hand your agent to a stranger.

Your AI coding tools — Claude Code, Codex, Cursor — run unreviewed code plus instructions from every skill, MCP server, and agent config you adopt. That code is wired into a process holding your credentials, shell, and network. Adopting one is npm install with an agent attached — no lockfile, no review gate, no record of what it did.

THE FOUR GAPS AGENTSTACK CLOSES
Anything a repo declares can run
A clone stays inert until you trust its exact bytes; any edit re-gates it.
Nothing narrows or records agents
A machine policy no repo can loosen fences tools, secrets, egress — every brokered call is logged.
Every CLI spells setup differently
One reviewed manifest renders them all; secrets stay references, never values.
Agents wreck trees by accident
guard blocks rm -rf, git reset --hard, and .env reads before they run.
YOU CLIMB IN SIX STEPS — EACH PAYS OFF ON ITS OWN
1 Unify→ 2 Verify→ 3 Guard→ 4 Trust→ 5 Scale→ 6 Confine
Nothing later is required to keep the earlier wins. This tutorial walks all six, plus the policy engine and the audit trail.

LESSON 3 · STEP 1 — UNIFY

One manifest for every CLI

agentstack init scans your machine, imports the MCP servers your CLIs already have, and writes one reviewed manifest: .agentstack/agentstack.toml. Then apply --write renders it back into every CLI's native config. Edit once, applied everywhere.

Secrets get lifted, not stored. Any inline token found in an imported config becomes a ${REF} placeholder. The real value goes to a gitignored .env or the OS keychain — the manifest itself never holds a secret value.
▶ Run each command in order — this is a simulation, nothing touches your machine.
Scripting or CI? init --secrets skip writes only the manifest — no prompts, no token values.

LESSON 4 · STEP 2 — VERIFY

Doctor catches drift; you pick the truth

Drift = a mismatch between the manifest and what's on disk — someone hand-edited a config, or the manifest stopped declaring something. doctor flags it, and every warning names its exact fix. The decision is always the same question: which side holds the truth?

hand-edit is right → adopt
agentstack adopt pulls the edit into the manifest, lifting any inline secret on the way.
manifest is right → apply
agentstack apply --write re-renders the manifest over the stray edit.

LESSON 5 · STEP 3 — GUARD

Block the accidents before they land

agentstack guard install wires a check into each CLI's own pre-tool-use hook. When the agent is about to run something obviously destructive, the hook refuses before the command executes.

⛔ rm -rf
recursive deletes outside the workspace
⛔ git reset --hard
history-destroying git commands
⛔ .env reads
reads of secret and key files
Honest limit: guard is cooperative, not enforcement. It catches an agent's accidents, not a determined attacker — a CLI that ignores its own hooks bypasses it entirely. Kernel-level confinement comes in Step 6.
LESSON 6 · STEP 4 — TRUST

Cloned repos stay inert until you approve them

Register the gateway once (gateway connect --all --write) and every repo serves its capabilities through it — but a fresh clone is inert: no server spawns, no skill enters context, no secret resolves, until you run agentstack trust .

TRY THE TRUST GATE — CLICK THE ACTIONS
cloned→ INERT→ review→ TRUSTED
What trust means — exactly. Trust pins a SHA-256 consent digest of the manifest, overlay, and lockfile, and gates whether declared servers may load. It does not vouch that the code is safe, and it does not confine a server once it runs. Vet one capability in depth first with agentstack explain <name>.
LESSON 7 · STEP 5 — SCALE

One governed stack across projects and machines

Three tools carry your setup beyond a single project:

profilesNamed subsets of the manifest ("backend", "design"). use <profile> --write materializes only that profile's skills and prunes the rest it owns.
libraryYour central home at ~/.agentstack/lib/ — skills and server definitions projects reference by name instead of copying. lib sync carries it across machines.
search + addOne search over your library, the bundled catalog, and the official MCP Registry. add from <id> lifts its secrets and renders it to all your CLIs at once. Skills also install from any repo: add skill owner/repo.
Try before installing: agentstack try <skill> runs a skill without writing anything.
LESSON 8 · STEP 6 — CONFINE

Four postures, from advisory to kernel-enforced

Every tracked run prints a posture label — how strongly the effective policy is actually enforced. Climb the ladder as trust in the workload drops:

HOST / ADVISORYplain run — tracked and logged, guard hooks advise, nothing confined
HOST / PROTECTEDrun --locked — fail-closed pre-launch gates (trust, lock, policy) + frozen tool surface; still on your host
SANDBOX / PROXIEDrun --sandbox — Docker container + host-side egress proxy; a direct network route still exists
LOCKDOWN / ENFORCEDrun --lockdown — the container's only route out is the audited proxy. Strongest AgentStack ships.

LESSON 10 · MODES, TEAMS & UNDO

Where files live, how teams share, how anything reverts

You always commit the intent — manifest + lockfile. Where the rendered artifacts live is a per-project delivery mode, chosen during init and switchable any time (switching never changes what you trust or what policy allows):

static
default — rendered files on disk behind a managed .gitignore block. Zero moving parts, works with every CLI.
clean-at-rest
files exist only between session start and session end — the repo stays pristine at rest.
zero-files
nothing on disk — the gateway serves each trusted repo’s stack live over MCP.

Teams commit .agentstack/ and never credentials: each teammate stores their own values with secret set, then apply --write. A maintainer can sign the lockfile (detached ed25519) and teammates verify --pubkey it. In zero-files mode trust stays per person, per machine.

Limit: restore reverts agentstack’s own recorded writes — a file a server already deleted is not brought back.
