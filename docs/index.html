Cloning a repo shouldn't hand your agent to a stranger.

AgentStack puts everything your AI coding tools are allowed to run into one reviewed file. A cloned repo can't auto-activate any of it until you approve it — and what runs through the gateway is firewalled and logged, with every run labelled by how strongly that is actually enforced.

Four gaps in how agents adopt code

Each one is real today in every agent CLI — and each has one command that closes it.

Anything a repo declares can run
A clone stays inert until you approve its exact bytes. Any edit — even a git pull — drops it back to untrusted.
closed by agentstack trust .
Nothing narrows or records agents
A machine policy no repo can loosen fences tools, secrets and egress — and every brokered call lands in the audit log.
closed by gateway connect --all
Every CLI spells setup differently
One reviewed manifest renders into every CLI's native config. Secrets stay ${REF} placeholders, never values.
closed by agentstack apply --write
Agents wreck trees by accident
Hooks block rm -rf, git reset --hard and .env reads before they execute — the command never runs.
closed by agentstack guard install

One pipeline, six words

You write a manifest, the lockfile pins it, you trust the result, policy narrows what may run, the gateway carries it — and every call lands in the audit log.

manifest→ lockfile→ trust→ policy→ gateway→ audit log

Climb in six steps

Each step pays off on its own — nothing later is required to keep the earlier wins.
