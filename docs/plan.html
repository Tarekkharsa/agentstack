Design · working review document · 14 July 2026

Make the boundaries easier to trust—and harder to bypass.

This is the working implementation plan produced from the documentation and code review. Small claim corrections can land immediately. Changes to security behavior remain explicit decision gates: review the contract first, then implement and prove it.

Product direction

Keep AgentStack the capability control plane.

The durable product is not a general assistant or another agent framework. It is the portable security contract beneath many harnesses: define a capability surface, deliver it, constrain it, and preserve evidence. The public model should lead with three workflows.

01 · COMPILE

Intent → native configuration

Use the manifest as the reviewed source of truth and render it across supported harnesses.

02 · BROKER

Small live capability surface

Use the gateway and profile leases to discover, fence, enforce, and record capabilities at runtime.

03 · CONFINE

Process and network boundary

Use sandbox and lockdown when the process itself—not only its brokered tool calls—must be constrained.

SUPPORTING SYSTEMS

Integrity, packaging, and evidence

Locks, signatures, libraries, packs, reports, and optimization strengthen the three primary workflows.

Implementation sequence

Close live security gaps first; improve documentation in parallel.

Done · D1

Make the machine floor resilient

Implemented: validate and atomically persist the last-known-good machine policy input. A broken edit uses that exact policy; if none exists, capability-serving and execution paths fail closed with a repair command. Inspection and repair remain available. The fail-open core::machine_policy() convenience is deleted and every consumer—including all guard paths—uses the result-returning CLI provider, so silently substituting an empty policy is no longer representable.

Urgent B · D4

Close the route around the gateway

Under lockdown, allow the container to reach the model and approved general egress through the proxy, but never a declared MCP host directly, and reject literal-IP and non-TLS bypasses. Resolve the run's servers once and freeze that set for both classification and gateway dispatch; route MCP only through the token-gated relay. If gateway config cannot be installed and native config reliably shadowed, refuse to start the container—there is no rendered-config fallback under lockdown. This removes the direct-MCP bypass behind the qualified Tools claim and closes the audit gap. Independently reviewable; may proceed alongside D1. See the D4 contract below.

Parallel · docs

One claim, one source

Documentation work does not gate D1 or D4. The first claim-correction pass is committed; remaining work prevents recurrence.

  • Add Stable, Experimental, Design, Historical, and Archived status labels.
  • Generate published HTML from technical Markdown instead of maintaining duplicate prose.
  • Add claim tests for matrix cells, CLI examples, versions, adapter counts, and feature status.
  • Rename unreleased lease_freeze directly to lease_capture; there are no external users, so add no alias or migration shim.
Next · D2

Extend the existing capability grant

Start from the shipped sandbox ExecutionPlan and Gateway::from_plan. Extract and extend that boundary so native render and profile leases consume the same resolved identities, verified pins, trust state, effective policy, secret grants, delivery/lifetime, confinement posture, and evidence identity. Preserve working code; do not replace it with a parallel abstraction.

Then · D3

Pin declared local code

Add optional local integrity inputs for stdio servers and other executable assets. Include their digests in the lock, surface them during trust review, and have doctor warn—not silently claim coverage—when a local path is executable but unpinned.

Later · D5

Make the sandbox name match its posture

After D4 and real compatibility evidence, make confined networking the ordinary sandbox behavior and make the weaker bridge/proxy-only mode explicit. With no external users, optimize the final interface rather than preserving the current flag behavior for its own sake.

BOUNDARY

Cache policy input, not a project ruleset

The effective compiled ruleset depends on project policy and server names. Persist only the validated machine Policy plus cache schema and source digest; compile it with each project normally.

PLACEMENT

Centralize runtime loading

Add one CLI-owned machine-policy provider used by gateway, render/apply, guard, explain, and sandbox planning. Delete core::machine_policy(); retain machine_policy_health() as the parsing/health surface. Keep the policy crate pure and preserve the existing dependency graph.

PERSISTENCE

Atomic, digest-guarded, and secret-free

Write a versioned, permission-restricted snapshot under AgentStack state using temp-file plus rename. Store the source-manifest digest and skip the write when it is unchanged, so normal policy reads remain read-only. The snapshot contains policy patterns only—never resolved secret values. No new dependency is required.

CEILING

Resilience, not tamper-proofing

The snapshot is user-writable like the machine manifest. It prevents accidental policy disappearance; it does not defend against a host-compromised agent acting as the user.

Files and tests before code: centralize the loader in the CLI, delete the fail-open core convenience, retain core parsing/health reporting, migrate all runtime and guard call sites to return errors instead of default policy, and add absent/valid/digest-unchanged/rotted-with-cache/rotted-first-run/corrupt-cache/future-version tests plus a gateway integration proving a cached deny survives manifest corruption. Doctor must distinguish UNCONFIGURED, DEGRADED, and BLOCKED. The policy intersection proptests remain mandatory.

D4 · concrete implementation contract

One MCP route under lockdown; the gateway is the sole authority for declared endpoints.

Under --lockdown the container's only peer is the dual-homed sidecar: an egress proxy for general traffic and a fixed-destination relay to the host gateway. The host gateway reaches upstream MCP servers host-side, so the relay path never crosses the container egress proxy. D4 denies the container's direct route to declared MCP hosts—leaving the relay the sole path—and hardens the residual escape hatches so the claim is precise. Scope is lockdown only; --sandbox keeps its labelled open route.

RESOLVED-RUN INPUT

Resolve servers once, freeze, share

The plan stores compiled policy and the manifest dir, not resolved servers—Gateway::from_plan re-resolves later, a classification/dispatch mismatch and TOCTOU. Add one profile-fenced resolved-server set: verify library pins, keep unresolved ${REF} placeholders, derive hosts from it, and hand the same frozen set to the gateway.

CLASSIFICATION

Declared HTTP hosts only

Every server of ServerType::Http—regardless of adapter transport spelling—contributes its normalized host (lowercased, trailing dot stripped, all ports). stdio is inherently gateway-only. TLS hides the path, so a host shared with a non-MCP service is denied whole: a documented, honest over-broad edge.

ONE ROUTE

Gateway config or refuse to start

No rendered-config fallback under lockdown. If the gateway entry cannot be installed and every native project config reliably shadowed, or an adapter cannot be mapped/shadowed, fail startup. An untrusted bundle may still run with zero MCP once its native config is scrubbed; an empty server set is benign but stale native config is still shadowed.

FIELD

gateway_only_hosts on the ruleset, v3

Add a canonical sorted, deduplicated host set to CompiledRuleset and bump RULESET_VERSION 2 → 3. Additive in shape but semantically incompatible—an older sidecar that ignored it would fail open. Evaluated in EgressGuard::decide before ordinary egress policy; only hosts enter ruleset.json, never URL credentials or secrets.

Boundary claim (precise): AgentStack blocks every normalized endpoint declared in the frozen run plan, and rejects literal-IP and non-TLS bypasses; it does not discover every alias the upstream service operates. The lockdown Tools cell reads enforced with exactly this scope—not an unqualified promise that a determined agent cannot reach a service by an undeclared name.

Fail closed & prove it: partial resolution or a pin failure fails the whole lockdown run rather than omitting a server whose direct endpoint might stay reachable; an unclassifiable declared HTTP endpoint refuses startup; version skew already denies all. Witnesses: a Docker-gated run where a direct CONNECT to the upstream is blocked while the same call succeeds and is recorded through the relay; unit tests for gateway-only precedence over an egress allow, host-exact/port-agnostic matching, literal-IP and non-TLS rejection, HTTP-vs-stdio classification, and the v3 field failing closed on an older consumer.

Required proof

Every changed promise needs an executable witness.

POLICY ROT

Last-known-good survives a bad edit

A valid deny remains effective after the machine manifest becomes malformed; first-run malformed state refuses activation and names the repair.

PATH CONSISTENCY

Every delivery path consumes the same grant

Property and integration tests prove policy can only narrow and no path reconstructs authority independently.

LOCAL CODE

A pinned script edit changes identity

A one-byte change fails lock verification and returns the project to review; an intentionally unpinned script is labelled as such.

ROUTE CLOSURE

Direct MCP is unreachable in lockdown

A direct connection to every declared upstream endpoint—and to literal-IP or non-TLS bypasses—fails under lockdown, while the same tool call succeeds through the gateway and is recorded.

DOC CLAIMS

Published matrices cannot drift

The website table is generated or tested against the same structured facts used by the authoritative specification.

EXAMPLE EVIDENCE

Recipes and proofs stay distinguishable

Examples identify whether they are tested fixtures, Docker-dependent reproductions, validated manifests, or illustrative snippets.

Review rule: D1–D5 should each be its own implementation and review unit. They touch distinct security boundaries and should not be bundled with documentation, catalog transport, or unrelated guard changes.

Non-goals

What this plan deliberately does not turn AgentStack into.

No personal memory platform, scheduler, background job system, general OAuth product, prompt-inspection firewall, or claim that approved destinations are safe. AgentStack remains the control plane beneath assistants, not the assistant itself. The composed agentstack run --locked hero UX and sequence-anomaly detection heuristic remain tracked in the roadmap; they are intentionally out of scope for this enforcement-boundary plan, not silently dropped.
